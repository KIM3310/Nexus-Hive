# Service Package - Nexus-Hive

This package defines the most practical service shape for this repository. It is public-safe: no price promises, financial projections, or buyer-specific contract assumptions are published here.

## Buyer and Wedge

- **Service lane:** Governed analytics workflow
- **Primary buyer:** Data platform teams, BI owners, and operations leaders with uncontrolled query risk
- **Front-door offer:** NL-to-query governance pilot
- **Service outcome:** Package safe analytics as a workflow with approvals, rejected-query review, and audit evidence.

## Build and Polish Priorities

- Clarify allowed data sources and query boundaries
- Expose approval and rejection states in the demo narrative
- Add handoff-ready analytics governance checklist

## Buyer-Ready Deliverables

- Workflow audit
- Query policy review
- Governed dashboard pilot

## Activation Path

1. Use the public demo, README, screenshots, and tests to establish proof.
2. Confirm the buyer workflow, data boundary, owner, and acceptance criteria.
3. Build only the smallest workflow needed to prove the service outcome.
4. Add cloud resources, accounts, storage, or paid integrations only after the workflow boundary is approved.

## Margin and Risk Controls

- Reuse policy templates and avoid custom warehouse work until buyer data boundaries are approved.
- Keep the first service scope narrow, repeatable, and evidence-led.
- Keep private financial planning outside the public repository.
- Keep logs, analytics, and reports free of private payloads unless explicit approval exists.

## Portfolio Signal

The portfolio should present this repository as **NL-to-query governance pilot** under the **Governed analytics workflow** lane, with proof links routed to the live demo or repository evidence.
